Selva

Introduction

The purpose of Selva is to keep your UI updated with your application data.

It uses only new attributes and a placeholder syntax close to JavaScript syntax, to keep things homogeneous.

Basics

Selva uses a new class of attributes: selva-*.

Those attributes will target the matched HTML attribute: selva-style will change the style attribute, selva-src the src attribute, etc.

Only one exception: selva-value: this will handle any type of content (value, checked, innerHTML), and bind the JS value to the element if it's an input.

The application variables can be either global ones or namespaced through an object.

selva-value="{myApplication.value}" and selva-value="{value}" will both work.

Selva can take {...} placeholders, where it will evaluate what is inside the brackets and replace it with the result.

For example, if you have a color variable, and you add a selva-style="color: {color}" to an element, your element will always have its color updated with the application variable.

If you add selva-value="{color}" to an color input, Selva will bind the input to the color value.

Multilingual applications

With Selva, you can easily switch languages in your app.

First, add a selva-language-folder attribute to the body tag, containing the path to the language folder.

This folder must contain JSON files named as follow: <language>.json.

Each JSON file must have a simple structure key -> text.

Keys must be the same over the JSON files, only the text value should change.

Then simply add the selva-language attribute to the body tag, Selva will automatically load the correct file in the folder.

Finally, use the selva-text attribute wherever you need to display some translated text.
